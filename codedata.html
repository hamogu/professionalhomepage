Codes and data

In a commitment to open science I aim to make all code and all data that I use for my research openly and freely available. I am convinced that this will in the long run benefit science in general, because other researchers (i.e. you) can use the models I developed for your own observations, reproduce my results and use my observations in different contexts.

My goal is to make codes and data available on a persistent and long-term basis. Thus, most of this is stored in external, institutionalized archives.

Data

Data in refereed articles

The first place to look for my data is in the published articles. Where tables exceed a few lines, you will find a machine-readable table on the publisher's website or at CDS, e.g. for the source table of all sources in IRAS 20050+2720.

Data in telescope archives

Much of my work is based on observations with large space telescopes, such as Chandra, XMM-Newton and the HST. The original datasets can be obtained from the archives curated by these missions.

Dataverse

Data that does not fall into the above categories is archived by the Astronomy Dataverse Network where it is also indexed and (in the future) will be discoverable using virtual observatory protocols or links from ADS.

Use this external link to get to my data. Specifically, you will find simulated X-ray spectra of accretion shocks in CTTS, ready for fitting your X-ray observation.

Codes

Published in a refereed article

Some of my code is published in the refereed article and is available at CDS (see this CDS link for the code to weight several 1D spectra of an inhomogeneously irradiated binary).

github

I organize my code using the git versioning system, and I use github to share the code.

My personal github repository contains projects started by me and copies (called forks in git) of larger projects where I have contributed code. The HST/COS line spread function convolution model is now also included in the distribution of the Sherpa fitting package.

Code for the YSOVAR project is collected in a separate repository.

I contribute to several larger community projects like astropy and the introduction to python at python4astronomers.
